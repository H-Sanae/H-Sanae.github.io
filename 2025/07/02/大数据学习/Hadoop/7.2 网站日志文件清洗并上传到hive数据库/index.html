相关总结

一、参数接收与文件校验

1. 用户输入获取

从命令行接收日期参数（格式：yyyymmdd）

2. 参数格式验证

检查输入是否符合 8 位纯数字格式

3. 文件存在性检查

验证对应日期的日志文件是否存在

二、日志数据清洗（ETL）

1. 日志文件读取

逐行读取原始日志文件

2. 基础字段提取

从日志行中解析 IP、日期和 URL

3. User-Agent 解析

从 User-Agent 字符串中提取操作系统和浏览器信息

4. 清洗数据写入

将提取的字段写入临时文件，使用制表符分隔

三、数据导入 Hive 数据库

1. Hive 表创建

检查目标表是否存在，不存在则创建分区表

2. 数据加载到分区

将清洗后的临时文件数据加载到对应日期分区

3. Hive 连接管理

建立与 HiveServer2 的连接并执行 SQL
